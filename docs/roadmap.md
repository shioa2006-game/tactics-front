# 開発ロードマップ

## プロジェクト概要

32×32マップ上で行うターン制戦略ゲーム。プレイヤーとCPUがそれぞれ4種類のユニットを使って陣地を争奪し、ターン終了時のポイントで勝敗を決める。ゲームバランス調整を目的とした自動対戦テストシステムを並行して開発する。

## Phase 1: MVP

**目標**: 基本ゲームが最初から最後まで動作すること

### ゲーム基本ルール（Phase 1で確定）

- ターン上限: 50ターン
- 勝利条件: 規定ターン終了時のポイント数で勝敗決定
- ポイント獲得: 陣地を完全占拠したターンにポイント獲得
- 陣地ルール: 9マス（3×3）の陣地に敵が1マスでもいると占拠無効

### ゲーム実装

- 32×32基本マップ生成
- ユニット2種（Knight、Scout）の基本実装
- 基本移動・攻撃システム
- シンプルなターン制
- 基本UI（Canvas描画）
- 手動プレイ可能な状態

### AI実装

- RandomAI: 完全ランダム行動
- GreedyAI: 1手先読み（最も近い目標への移動）
- 基本対戦システム
- 簡易ログ出力

### 成功指標

- ゲーム完走率: 100%
- AI思考時間: 1秒以内
- 基本ルール動作: 正常

### 状態

完了（docs/phase1_actual.md 参照）

## 設計上の注意点（全Phaseを通じて）

- ゲームロジックと描画は必ず分離する（Web Worker対応のため）
- ロジック層はピュアJSで実装し、描画ライブラリに依存しない
- Phase 1から「テスト可能な構造」を意識して設計する
- 各Phaseの完了時にREADMEの「現在のPhase」を更新する
