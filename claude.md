# p5.js ブラウザゲーム開発

## プロジェクト概要
p5.jsを使用したブラウザゲームの開発プロジェクト。

## 技術スタック
- p5.js（CDN読み込み）
- HTML / CSS / JavaScript
- 外部フレームワーク（React, Vue等）は使用しない

## 言語・命名規則
- コメント・説明・UIテキストはすべて日本語
- 関数名・変数名は英語キャメルケース（例: playerScore, moveEnemy）
- 定数は大文字スネークケース（例: MAX_LIVES, GRAVITY）
- マジックナンバーは定数化する

## コード設計ルール
- 1関数は30行以内を目安に、長くなったら分割する
- setup(): 初期化処理のみ。描画ロジックを入れない
- draw(): 描画とゲームループ。重い計算は別関数に切り出す
- グローバル変数はゲーム状態管理に必要な最小限にする
- 関連する状態はオブジェクトにまとめる（例: let game = { score: 0, lives: 3 }）
- 処理の意図がわかるコメントをつける（何をしているかではなく、なぜそうしているか）

## ファイルの扱い
- ファイルの読み書きはUTF-8で行う
- PowerShell環境では Set-Content -Encoding UTF8 を使用する
- assets/ 内の素材ファイルは勝手に削除・リネームしない

## コード変更時のルール
- 既存の動作している機能を壊さないことを最優先にする
- 指示された範囲だけを変更する。関係ない箇所のリファクタリングを勝手にしない
- 変更の意図が明確でない大規模な書き換えはしない
- 未使用コードは削除する

## やってはいけないこと
- p5.js以外のフレームワークやゲームエンジンを導入しない
- npm / webpack / bundler 等のビルドツールを導入しない
- 動作確認なしに複数の機能を同時に変更しない
- 既存のゲームバランスに関わる数値を説明なく変更しない
